  Issue #8 — Add default-suppressed rule tier

  Why: Several rules in Issue #7 (S049 name-not-gerund, S047 body-no-examples) are too opinionated to ship as default-warn. agent-lint has 3 tiers
  (error/warn/suppressed); agnix currently has 2. Without a suppressed tier, these rules either spam by default or need manual disabling after port.

  What: Extend severity model in crates/agnix-core/src/config.rs:
  - Add Severity::Suppressed variant (rule is defined and loadable but does not fire unless explicitly enabled in .agnix.toml).
  - Update rule loading from rules.json to respect a new "default_state": "suppressed" field.
  - Update .agnix.toml [rules] to allow enabling suppressed rules (e.g., enabled = ["CC-SK-049", ...]).
  - Update crates/agnix-lsp to respect the new state on config change.

  Risk: Touches config model; requires maintainer design input. Open as an RFC-shaped issue first.

  Dependency: S047 and S049 in Issue #7 ship at default-warn until #8 lands; re-classify as suppressed after.
